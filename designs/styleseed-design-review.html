Α. Καθολικά (design system)

HIGHΔιπλό token: --oxblood = --orange = #8a6a1c styles.css:1 Δύο ονόματα, μία τιμή. Ο κώδικας «νομίζει» ότι έχει τρίτο χρώμα (toast, personal-bests, progress-alert χρησιμοποιούν oxblood) ενώ στην πράξη είναι το orange. Επιπλέον gold #c9a227 και orange #8a6a1c εναλλάσσονται ως accent χωρίς κανόνα.
Κράτα τον κανόνα που ήδη σχεδόν ισχύει και γράψε τον ως token: orange = accent πάνω σε χαρτί, gold = accent πάνω σε μελάνι (σκούρο φόντο). Σβήσε το --oxblood ή δώσε του πραγματική τιμή (π.χ. βαθύ κόκκινο-καφέ #7a3325 για destructive/PR). 
:root {
  --accent: #8a6a1c;        /* σε ανοιχτό φόντο (πρώην orange/oxblood) */
  --accent-on-ink: #c9a227; /* σε σκούρο φόντο (πρώην gold) */
  --danger: #7a3325;        /* διαγραφές, λάθη */
}
HIGHHardcoded hex όπου υπάρχει (ή λείπει) token styles.css:14, 236-241, 263, 281-285 κ.ά. Το #3d3828 (γραμμές πάνω σε ink) επαναλαμβάνεται ~15 φορές· επίσης #b9af96, #a89f86, #cfc6ac, #918873, #625b48, #8f866e — όλα «muted πάνω σε μελάνι» σε 6 παραλλαγές. Το glow του reward ring χρησιμοποιεί #ffd447 (styles.css:319) — τέταρτη χρυσή απόχρωση.
Τρία νέα tokens και μαζική αντικατάσταση — μειώνει τις «σκόρπιες» αποχρώσεις από ~10 σε 3: 
:root {
  --ink-line:  #3d3828;  /* διαχωριστικά σε σκούρο φόντο */
  --ink-muted: #a89f86;  /* δευτερεύον κείμενο σε σκούρο φόντο */
  --ink-faint: #6f695a;  /* micro-labels σε σκούρο φόντο */
}
Το reward glow να γίνει var(--accent-on-ink) αντί για #ffd447.
HIGHContrast: το --muted #7c745f σε micro-μεγέθη πέφτει κάτω από AA styles.css:1 + χρήσεις σε .55–.68rem (98, 127, 180, 384) #7c745f πάνω σε #efe8d8 ≈ 3.9:1 — κάτω από το 4.5:1 που απαιτεί το μικρό κείμενο, και χρησιμοποιείται σχεδόν αποκλειστικά σε πολύ μικρό uppercase (labels, hints, footers).
--muted: #6d6551 (≈4.8:1) — οπτικά σχεδόν ίδιο, περνάει AA. Σε σκούρο φόντο το #8f866e των day-tiles (styles.css:129) οριακά περνάει, αλλά ανέβασέ το στο --ink-muted για σιγουριά.
MEDΚαμία τυπογραφική κλίμακα — και μεγέθη κάτω από όριο αναγνωσιμότητας styles.css:284 (.42rem), 224 (.48rem), 281 (.5rem), 379 (.56rem) … Μετρήθηκαν 20+ διαφορετικά μεγέθη μεταξύ .42 και .9rem. Το .42rem = 6.7px («VERSION» στο μενού) και το .48rem = 7.7px (PR badge) είναι πρακτικά μη αναγνώσιμα — και σε 800 weight uppercase με letter-spacing γίνονται μουτζούρα σε κινητό.
Κλίμακα 6 βημάτων για τα micro/label μεγέθη με δάπεδο τα 10px: 
:root {
  --fs-micro: .625rem;  /* 10px — stamps, version, badges (πρώην .42–.58) */
  --fs-label: .7rem;    /* 11px — labels, eyebrows (πρώην .6–.68) */
  --fs-meta:  .78rem;   /* 12.5px — hints, meta (πρώην .7–.76) */
}
Ό,τι σήμερα είναι <.58rem ανεβαίνει στο --fs-micro. Το «στάμπα» ύφος διατηρείται — απλώς διαβάζεται.
MEDΤρεις «γλώσσες» σκιών χωρίς κανόνα styles.css:26 (soft blur), 67 (hard offset 5px 5px 0), 128 (drop-shadow), 325 (glow) Hard offset στα κουμπιά/panels, soft blur στις «χάρτινες» κάρτες, drop-shadows στα βέλη, glow στα rewards. Καθεμία μόνη της δουλεύει· μαζί χωρίς κανόνα διασπούν τη συνοχή.
Γράψε τον κανόνα που ήδη υπονοείται και εφάρμοσέ τον 100%: φυσικά/χάρτινα αντικείμενα (κάρτες home, session page, info-panel) = μαλακή σκιά με blur · UI controls (κουμπιά, panels, dialogs) = hard offset χωρίς blur. Παράβαση: το .confirm-dialog ορίζει box-shadow:14px 14px 0 var(--orange) και αμέσως μετά το μηδενίζει (styles.css:133→134 — νεκρός κώδικας, διάλεξε το ένα).
LOWCSS αρχιτεκτονική: στρώματα-«εποχές» που αλληλοαναιρούνται styles.css — .avatar-figure ορίζεται 3 φορές (262, 269, 274)· .profile-card 3 φορές (263, 296, 310)· .routine-manager/.routine-card 3 φορές (118, 390, 410+) Κάθε redesign προστέθηκε από κάτω αντί να αντικαταστήσει το προηγούμενο. Λειτουργεί, αλλά κάθε μελλοντική αλλαγή απαιτεί ανάγνωση 3 ορισμών, και τα «νεκρά» πρώτα στρώματα (π.χ. avatar-head/body/legs των 262) δεν χρησιμοποιούνται πια.
Πέρασμα συγχώνευσης: ένας ορισμός ανά component, σβήσιμο των νεκρών (avatar full-body σύστημα, παλιό .routine-manager dark grid, πρώτο .profile-card). Καμία οπτική αλλαγή — μόνο μείωση ~20% του αρχείου και μία πηγή αλήθειας.

Β. States & Feedback toast, empty, errors

HIGHΈνα toast για όλα — τα λάθη «σφραγίζονται» ως RECORDED styles.css:231-232, app.js:784, 1163, 1255, 1276 Το toast έχει μόνιμο prefix content:"LOGBOOK / RECORDED". Όταν όμως καλείται με μήνυμα λάθους («Η ημερομηνία … δεν μπορεί να είναι μεταγενέστερη», «Δεν υπάρχει αρκετός χώρος για την εικόνα»), ο χρήστης βλέπει τη σφραγίδα επιτυχίας πάνω από το λάθος — αντιφατικό σήμα.
Δεύτερο όρισμα στη toast(message, kind) και δύο παραλλαγές: 
function toast(message, kind = 'recorded') {
  const el = $('#toast');
  el.textContent = message;
  el.classList.toggle('toast-error', kind === 'error');
  el.classList.add('show');
  …
}
.toast::before { content:"LOGBOOK / RECORDED"; }
.toast.toast-error { background:var(--danger); border-left-color:var(--ink); }
.toast.toast-error::before { content:"LOGBOOK / ΠΡΟΣΟΧΗ"; }
Κλήσεις λάθους (app.js:1163, 1255, 1276, 1320, 1347-1357, 1474) περνούν 'error'.
MEDEmpty states χωρίς επόμενη ενέργεια app.js:673 (Ιστορικό), 726 (Επίβλεψη), 897 (κενό πρόγραμμα) Η φωνή είναι εξαιρετική («Η γραμμή εκκίνησης είναι εδώ.») αλλά σταματά εκεί — ο χρήστης πρέπει μόνος του να βρει πού πάει. Το empty της Επίβλεψης (l.726) είναι σκέτο κείμενο χωρίς καν τίτλο, ασύμβατο με τα άλλα δύο.

'<div class="empty"><strong>Η γραμμή εκκίνησης είναι εδώ.</strong>

Δ. Motion & reduced-motion

MEDΤο prefers-reduced-motion καλύπτει ~μισά animations styles.css:143, 354, 407, 433, 472 (καλύπτονται) — ΔΕΝ καλύπτονται: 130 (week-shift), 128 (arrows), 131 (session-card transform), 105 (toast), 135 (slide/reveal keyframes), 3 (scroll-behavior) Οι μεγαλύτερες κινήσεις (wipe, dialog) προστατεύονται, αλλά week-strip slides, card translations και smooth scroll τρέχουν πάντα.
Ένας καθολικός guard στο τέλος του αρχείου αντί για ad-hoc εξαιρέσεις: 
@media (prefers-reduced-motion: reduce) {
  html { scroll-behavior:auto; }
  *, *::before, *::after {
    animation-duration:.01ms !important;
    animation-iteration-count:1 !important;
    transition-duration:.01ms !important;
  }
}
Και σβήσιμο των 5 σκόρπιων guards — λιγότερος κώδικας, πλήρης κάλυψη.
