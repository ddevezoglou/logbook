# Logbook — Ημερολόγιο Προπόνησης

Το **Logbook** είναι μια local-first εφαρμογή καταγραφής προπονήσεων, οργάνωσης μικρόκυκλων και παρακολούθησης της προόδου. Το project βρίσκεται σε ενεργή ανάπτυξη και η τρέχουσα έκδοση του interface είναι η **0.1.0**.

## Τρέχουσα κατάσταση

Η εφαρμογή λειτουργεί στον browser και διαθέτει πλέον σύνδεση λογαριασμού μέσω Supabase. Τα προπονητικά δεδομένα εξακολουθούν να αποθηκεύονται τοπικά με `localStorage`, επομένως παραμένουν στον συγκεκριμένο browser και στη συγκεκριμένη συσκευή μέχρι να ολοκληρωθεί ο συγχρονισμός.

### Τι περιλαμβάνει σήμερα

- Αρχική σελίδα με γρήγορη πλοήγηση, προσωπική κάρτα αθλητή και καθημερινό quote από το επεξεργάσιμο `quotes.js`.
- Καταγραφή προγραμματισμένης ή ελεύθερης προπόνησης, με ασκήσεις, σετ, επαναλήψεις, βάρος, τεχνικά cues και σχόλια.
- Πολλαπλοί ανεξάρτητοι μικρόκυκλοι 3–10 ημερών, προαιρετικά με εμφάνιση ημερών εβδομάδας, και ένας ενεργός μικρόκυκλος που αντιστοιχίζει κάθε ημερομηνία στη σωστή προπόνηση.
- Επεξεργασία προγράμματος και δυνατότητα συγχρονισμού αλλαγών ονομάτων ή ασκήσεων με το υπάρχον ιστορικό.
- Ιστορικό προπονήσεων με συχνότητα 7 ημερών, συνολικά στατιστικά, προσωπικά καλύτερα και αναλυτική προβολή κάθε καταγραφής.
- Επίβλεψη προόδου ανά προπόνηση, άσκηση και σετ, με σύγκριση βάρους και επαναλήψεων μεταξύ καταγραφών.
- Προφίλ αθλητή με βασικά στοιχεία, έτοιμο ή προσαρμοσμένο avatar και σύστημα επιβράβευσης συνέπειας ανά ενεργό πρόγραμμα.
- Πολυγλωσσικό interface σε Ελληνικά, Αγγλικά, Γαλλικά και Γερμανικά.
- Εγγραφή, σύνδεση και αποσύνδεση λογαριασμού μέσω Supabase Auth με email/κωδικό ή Google, χωρίς να επηρεάζονται οι τοπικές προπονήσεις.
- Responsive σχεδιασμό για κινητό και υπολογιστή.
- Αυτόματη μεταφορά παλαιότερων δεδομένων από το προηγούμενο ενιαίο μοντέλο προγράμματος/καταγραφών.
- Αυτοματοποιημένα tests για τις βασικές ροές, τις μετατροπές δεδομένων και επιβεβαιωμένα regression bugs.

## Εκτέλεση στον υπολογιστή

Δεν απαιτείται build. Μπορείς να ανοίξεις απευθείας το `index.html` ή, για πιο αξιόπιστη τοπική λειτουργία, να τρέξεις μέσα από τον φάκελο:

```powershell
npx.cmd serve .
```

και να ανοίξεις τη διεύθυνση που θα εμφανιστεί στο terminal.

## Tests

Την πρώτη φορά εγκατέστησε τις εξαρτήσεις και μετά εκτέλεσε τη σουίτα ελέγχων:

```powershell
npm.cmd install
npm.cmd test
```

## Cloud development

Η πρώτη υποδομή του Supabase βρίσκεται στα `supabase-config.js`, `supabase-client.js`, `auth.js` και `supabase/migrations/`. Η εφαρμογή υποστηρίζει εγγραφή και σύνδεση με email/κωδικό ή Google, διατήρηση συνεδρίας και αποσύνδεση, αλλά εξακολουθεί να χρησιμοποιεί το `localStorage` ως πηγή αλήθειας για τα προπονητικά δεδομένα. Δεν γίνεται ακόμη αυτόματη μεταφορά ή συγχρονισμός τους.

Το αρχικό migration δημιουργεί τους πίνακες `profiles`, `routines` και `sessions`, ενεργοποιεί Row Level Security και περιορίζει κάθε εγγραφή στον συνδεδεμένο χρήστη. Εφαρμόζεται στο development project με το Supabase CLI, ώστε το remote migration history να παραμένει συγχρονισμένο με τα versioned αρχεία του repository.

## Περιορισμοί της έκδοσης 0.1.0

- Ο λογαριασμός χρήστη και η online βάση υπάρχουν, αλλά οι προπονήσεις δεν συγχρονίζονται ακόμη μεταξύ συσκευών.
- Δεν υπάρχει ακόμη ασφαλές export/import ή αυτόματο backup των δεδομένων.
- Η εφαρμογή προσαρμόζεται σε οθόνη κινητού, αλλά δεν εγκαθίσταται ακόμη ως PWA ή native mobile app.
- Οι ασκήσεις καταχωρίζονται ως ελεύθερο κείμενο και δεν συνδέονται ακόμη με ενιαία προσωπική βιβλιοθήκη.

- [ ] **Λειτουργική ολοκλήρωση της ενότητας «Επίβλεψη».** Εξέλιξη της υπάρχουσας ενότητας σε πληρέστερο εργαλείο προπονητικής αξιολόγησης, με αξιόπιστη επιλογή και σύγκριση δεδομένων, περισσότερες χρήσιμες μετρικές και σαφή αποτύπωση της προόδου ανά προπόνηση, άσκηση και σετ, χωρίς αλλαγή της ολοκληρωμένης αισθητικής κατεύθυνσης.

- [ ] **Ολοκλήρωση της μετάβασης σε Supabase.** Το Authentication, το αρχικό schema και το Row Level Security έχουν στηθεί. Απομένουν ο συγχρονισμός πολλών συσκευών, η ασφαλής μεταφορά από `localStorage` και η διαδικασία ανάκτησης δεδομένων.
- [ ] **Offline-first συγχρονισμός.** Η καταγραφή να συνεχίζει χωρίς σύνδεση και να συγχρονίζεται με ασφάλεια όταν επανέρχεται το δίκτυο, χωρίς διπλότυπα ή απώλειες.

- [ ] **Installable PWA.** Manifest, service worker, offline shell, εικονίδια και σωστή συμπεριφορά εγκατάστασης σε Android/iOS.

## Κατεύθυνση

Η βασική αρχή είναι να παραμείνει η καταγραφή γρήγορη και αξιόπιστη. Κάθε επόμενο βήμα πρέπει να διατηρεί τη λειτουργία χωρίς σύνδεση, να προστατεύει το υπάρχον ιστορικό και να αποφεύγει περιττή πολυπλοκότητα την ώρα της προπόνησης.
