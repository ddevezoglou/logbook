# Logbook — Ημερολόγιο Προπόνησης

Το **Logbook** είναι μια local-first εφαρμογή καταγραφής προπονήσεων, οργάνωσης μικρόκυκλων και παρακολούθησης της προόδου. Το project βρίσκεται σε ενεργή ανάπτυξη και η τρέχουσα έκδοση του interface είναι η **0.3.2**.

## 📱 Εγκατάσταση στο κινητό

### [Άνοιξε το Logbook](https://ddevezoglou.github.io/logbook/)

Το repository του GitHub περιέχει τον κώδικα· η εφαρμογή ανοίγει από τον παραπάνω σύνδεσμο.

2. Πάτησε το μενού `⋮`.
3. Επίλεξε **Εγκατάσταση εφαρμογής** ή **Προσθήκη στην αρχική οθόνη**.

**iPhone — Safari**

2. Πάτησε **Κοινοποίηση** (το τετράγωνο με το επάνω βέλος).
3. Επίλεξε **Προσθήκη στην οθόνη Αφετηρίας** και μετά **Προσθήκη**.

> Η εγκατάσταση θα είναι διαθέσιμη μετά το πρώτο επιτυχημένο deployment στο GitHub Pages.

## Τρέχουσα κατάσταση

Η εφαρμογή λειτουργεί στον browser με υποχρεωτική σύνδεση λογαριασμού και συγχρονισμό μέσω Supabase. Στην εκκίνηση ελέγχεται πρώτα η συνεδρία, εμφανίζεται αποκλειστικά η οθόνη σύνδεσης όταν δεν υπάρχει login και το κύριο UI φορτώνεται μόνο μετά τον αρχικό συγχρονισμό. Το `localStorage` παραμένει η άμεση local-first αποθήκευση, ενώ προγράμματα, προπονήσεις, προφίλ και ρυθμίσεις συγχρονίζονται αυτόματα σε ένα versioned cloud snapshot ανά λογαριασμό.

### Τι περιλαμβάνει σήμερα

- Αρχική σελίδα με γρήγορη πλοήγηση, προσωπική κάρτα αθλητή και καθημερινό quote από το επεξεργάσιμο `quotes.js`.
- Καταγραφή προγραμματισμένης ή ελεύθερης προπόνησης, με ασκήσεις, σετ, επαναλήψεις, βάρος, τεχνικά cues και σχόλια.
- Πολλαπλοί ανεξάρτητοι μικρόκυκλοι 3–10 ημερών, προαιρετικά με εμφάνιση ημερών εβδομάδας, και ένας ενεργός μικρόκυκλος που αντιστοιχίζει κάθε ημερομηνία στη σωστή προπόνηση.
- Επεξεργασία προγράμματος και δυνατότητα συγχρονισμού αλλαγών ονομάτων ή ασκήσεων με το υπάρχον ιστορικό.
- Ιστορικό προπονήσεων με συχνότητα 7 ημερών, συνολικά στατιστικά, προσωπικά καλύτερα και αναλυτική προβολή κάθε καταγραφής.
- Επίβλεψη προόδου ανά προπόνηση, άσκηση και σετ, με σύγκριση βάρους και επαναλήψεων μεταξύ καταγραφών.
- Προφίλ αθλητή με βασικά στοιχεία, έτοιμο ή προσαρμοσμένο avatar και σύστημα επιβράβευσης συνέπειας ανά ενεργό πρόγραμμα.
- Πολυγλωσσικό interface σε Ελληνικά, Αγγλικά, Γαλλικά και Γερμανικά.
- Εγγραφή, σύνδεση και αποσύνδεση λογαριασμού μέσω Supabase Auth με email/κωδικό ή Google, αυτόματη πρώτη μεταφορά των τοπικών δεδομένων και συγχρονισμός μεταξύ συσκευών.
- Responsive σχεδιασμό για κινητό και υπολογιστή.
- Αυτόματη μεταφορά παλαιότερων δεδομένων από το προηγούμενο ενιαίο μοντέλο προγράμματος/καταγραφών.
- Αυτοματοποιημένα tests για τις βασικές ροές, τις μετατροπές δεδομένων και επιβεβαιωμένα regression bugs.

## Εκτέλεση στον υπολογιστή

Δεν απαιτείται build. Επειδή όμως η σύνδεση λογαριασμού χρησιμοποιεί ασφαλή authentication callbacks, μην ανοίγεις το `index.html` απευθείας ως `file://`. Τρέξε μέσα από τον φάκελο:

```powershell
npx.cmd serve . -l 3000
```

και άνοιξε το `http://localhost:3000/` όσο το terminal παραμένει ανοιχτό. Έτσι οι ροές Google login, επιβεβαίωσης email και αλλαγής κωδικού επιστρέφουν στη σωστή σελίδα αντί να καταλήγουν σε ανενεργό `localhost`.

## GitHub Pages και PWA

Το production deployment ορίζεται στο `.github/workflows/pages.yml` και δημοσιεύει μόνο τα runtime αρχεία της εφαρμογής στο GitHub Pages. Μετά την ενεργοποίηση του GitHub Pages με source **GitHub Actions**, κάθε push στο `main` δημιουργεί νέο production artifact. Τα tests, τα designs, τα scripts και τα seed εργαλεία δεν συμπεριλαμβάνονται σε αυτό.

Για το repository `ddevezoglou/logbook`, η αναμενόμενη διεύθυνση είναι `https://ddevezoglou.github.io/logbook/`. Η ίδια ακριβώς διεύθυνση πρέπει να προστεθεί στα επιτρεπόμενα Redirect URLs του Supabase Auth πριν δοκιμαστούν Google login, επιβεβαίωση email ή ανάκτηση κωδικού από το δημοσιευμένο app.

Το manifest, τα app icons, οι self-hosted γραμματοσειρές και το offline shell είναι έτοιμα για τη διαδρομή `/logbook/`. Το TODO του installable PWA παραμένει ανοικτό μέχρι να ολοκληρωθεί η πρώτη δημοσίευση και να επιβεβαιωθεί η εγκατάσταση σε πραγματικές συσκευές Android και iOS.

## Cloud development

Η υποδομή του Supabase βρίσκεται στα `supabase-config.js`, `supabase-client.js`, `auth.js`, `cloud-sync.js` και `supabase/migrations/`. Η πρώτη φόρτωση απαιτεί επιβεβαιωμένη συνεδρία και επιτυχημένο αρχικό sync. Μετά τη φόρτωση, οι αλλαγές γράφονται πρώτα τοπικά και συγχρονίζονται όταν υπάρχει δίκτυο. Το UI του λογαριασμού εμφανίζει την κατάσταση του sync και προσφέρει χειροκίνητο «Συγχρονισμός τώρα».

Τα migrations δημιουργούν τους πίνακες `profiles`, `routines`, `sessions` και το versioned `user_sync_state`, ενεργοποιούν Row Level Security και περιορίζουν κάθε εγγραφή στον συνδεδεμένο χρήστη. Εφαρμόζονται στο development project με το Supabase CLI, ώστε το remote migration history να παραμένει συγχρονισμένο με τα versioned αρχεία του repository.

## Περιορισμοί της έκδοσης 0.3.2

- Το sync είναι snapshot-based και όχι live collaborative editing. Σε ταυτόχρονες αλλαγές από δύο offline συσκευές διατηρούνται οι μοναδικές προπονήσεις/ρουτίνες και εφαρμόζεται optimistic conflict retry, αλλά δεν υπάρχει ακόμη οθόνη χειροκίνητης επίλυσης αλλαγών στο ίδιο αντικείμενο.
- Δεν υπάρχει ακόμη ασφαλές export/import ή αυτόματο backup των δεδομένων.
- Η υποδομή εγκατάστασης PWA είναι έτοιμη, αλλά δεν έχει ακόμη δημοσιευτεί και επιβεβαιωθεί σε πραγματικές συσκευές Android/iOS. Native mobile app δεν υπάρχει ακόμη.
- Οι ασκήσεις καταχωρίζονται ως ελεύθερο κείμενο και δεν συνδέονται ακόμη με ενιαία προσωπική βιβλιοθήκη.

- [ ] **Λειτουργική ολοκλήρωση της ενότητας «Επίβλεψη».** Εξέλιξη της υπάρχουσας ενότητας σε πληρέστερο εργαλείο προπονητικής αξιολόγησης, με αξιόπιστη επιλογή και σύγκριση δεδομένων, περισσότερες χρήσιμες μετρικές και σαφή αποτύπωση της προόδου ανά προπόνηση, άσκηση και σετ, χωρίς αλλαγή της ολοκληρωμένης αισθητικής κατεύθυνσης.

### Cloud και λογαριασμοί

- [x] **Αρχικός συγχρονισμός πολλών συσκευών.** Authentication, Row Level Security, ασφαλής πρώτη μεταφορά από `localStorage`, versioned cloud snapshot και αυτόματο retry όταν επανέρχεται το δίκτυο.
- [ ] **Εργαλεία ανάκτησης και προχωρημένων conflicts.** Ιστορικό snapshots, χειροκίνητη επαναφορά και UI επίλυσης όταν το ίδιο αντικείμενο αλλάξει ταυτόχρονα σε δύο offline συσκευές.
- [ ] **Export/Import δεδομένων.** Εξαγωγή προπονήσεων και προγραμμάτων σε JSON/CSV από το UI και ασφαλής επανεισαγωγή, ώστε ο χρήστης να έχει πάντα τα δεδομένα του.

### Mobile και διάθεση

- [ ] **Installable PWA.** Manifest, service worker, offline shell, εικονίδια και self-hosted γραμματοσειρές έχουν υλοποιηθεί. Απομένουν η πρώτη δημοσίευση στο GitHub Pages και η επιβεβαίωση εγκατάστασης/εκκίνησης σε πραγματικές συσκευές Android και iOS.
- [ ] **Offline boot με αποθηκευμένη συνεδρία.** Όταν δεν υπάρχει δίκτυο αλλά υπάρχει έγκυρη συνεδρία και τοπικά δεδομένα, το app να φορτώνει κατευθείαν με ένδειξη «Εκτός σύνδεσης» αντί για error στο gate· ο συγχρονισμός να εκτελείται αυτόματα όταν επανέλθει το δίκτυο (ο μηχανισμός `online` υπάρχει ήδη στο `cloud-sync.js`).

- [ ] **Ποιότητα έκδοσης.** End-to-end έλεγχοι των βασικών ροών σε πραγματικά mobile viewports, accessibility audit και ενιαία διαχείριση version/release notes.

### Τεχνικό χρέος και υποδομή

- [ ] **Error tracking.** Οι αποτυχίες sync καταλήγουν μόνο σε `console.warn` — προσθήκη ελαφριάς καταγραφής σφαλμάτων ώστε να φαίνονται προβλήματα πραγματικών χρηστών.
- [x] **Seed εργαλεία εκτός production.** Το GitHub Pages workflow δημιουργεί production artifact από ρητή λίστα runtime αρχείων και δεν συμπεριλαμβάνει τα `seed.html`/`seed-week.js`.
